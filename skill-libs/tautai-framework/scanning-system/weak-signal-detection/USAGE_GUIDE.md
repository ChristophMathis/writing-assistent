# Weak Signal Detection Skill - Complete Guide

## Overview

This skill supports organizational designers in establishing robust weak signal detection capabilities. It provides comprehensive frameworks, assessment tools, and implementation guidance for building the cultural, structural, and capability foundations needed for effective strategic foresight.

## What's Included

### 1. Main Skill File (SKILL.md)
The core guidance document covering:
- Complete workflow for capability development
- Step-by-step implementation guidance
- Cultural and structural interventions
- Detection methods overview
- Classification frameworks
- Integration with strategic processes
- Common challenges and solutions

### 2. Scripts (Executable Tools)

**assess_readiness.py**
- Comprehensive organizational readiness assessment
- Evaluates 4 dimensions: cultural, structural, cognitive, resource
- Calculates maturity level (1-5 scale)
- Generates prioritized recommendations
- Provides actionable next steps

**classify_signals.py**
- Signal classification framework
- Type categorization (technological, social, economic, etc.)
- Strength assessment (very weak to strong)
- Impact evaluation with weighted criteria
- Priority scoring (immediate action to track & review)
- Response option generation

## How to Use This Skill

### For Initial Assessment

1. **Run the readiness assessment**:
   - Use assess_readiness.py for automated scoring
   - Or walk through assessment_questionnaire.md for detailed evaluation
   - Identify maturity level and priority improvement areas

2. **Review cultural enablers**:
   - Read cultural_enablers.md to understand prerequisites
   - Assess which foundations are weak
   - Identify specific indicators present/absent

3. **Generate recommendations**:
   - The assessment script provides prioritized recommendations
   - Focus on 2-3 improvement areas initially
   - Address cultural factors before structural ones

### For Implementation

1. **Design interventions**:
   - Use SKILL.md Step 2 for intervention ideas
   - Customize based on organizational context
   - Start with quick wins to build momentum

2. **Select detection methods**:
   - Review detection_methods.md for full details
   - Choose 2-3 methods appropriate to context
   - Consider readiness level when selecting

3. **Establish processes**:
   - Use SKILL.md Step 5 for governance structures
   - Define roles and decision rights
   - Set up regular cadences (weekly/monthly/quarterly)

### For Signal Classification

1. **Use the classification framework**:
   - Run classify_signals.py for structured analysis
   - Or manually apply the framework from SKILL.md Step 4
   - Document signals systematically

2. **Prioritize signals**:
   - Assess impact across 5 criteria
   - Calculate priority score
   - Match response to priority level

3. **Track and monitor**:
   - Maintain signal database
   - Update strength and priority as signals evolve
   - Connect related signals

## Typical Usage Scenarios

### Scenario 1: Starting from Scratch
**Context**: Organization has no formal weak signal capability

**Approach**:
1. Conduct readiness assessment (assess_readiness.py)
2. Review cultural_enablers.md to understand gaps
3. Design Phase 1 interventions (months 1-6)
4. Focus on building psychological safety and initial scanning
5. Start with 1-2 simple detection methods
6. Demonstrate early value to build support

### Scenario 2: Strengthening Existing Capability
**Context**: Organization has informal practices, wants to formalize

**Approach**:
1. Assess current maturity level
2. Identify which dimensions need strengthening
3. Review detection_methods.md for advanced techniques
4. Implement classification framework (classify_signals.py)
5. Establish formal governance and processes
6. Integrate with strategic planning

### Scenario 3: Cultural Barriers
**Context**: Methods are known but culture blocks effectiveness

**Approach**:
1. Deep dive into cultural_enablers.md
2. Assess each cultural foundation carefully
3. Design targeted cultural interventions
4. Focus on leadership behavior change
5. Create safe spaces for signal sharing
6. Celebrate signal sharing regardless of outcome

### Scenario 4: Signal Overload
**Context**: Too many signals, hard to prioritize

**Approach**:
1. Implement prioritization framework (SKILL.md Step 4)
2. Use classify_signals.py for systematic scoring
3. Establish triage process
4. Focus on strategic relevance
5. Set clear escalation thresholds
6. Regular portfolio reviews to manage signal database

## Key Concepts

### Weak Signals
Early, ambiguous indicators of emerging change with potential strategic impact. Characteristics:
- Low signal-to-noise ratio
- Appear at organizational periphery
- Often contradict current assumptions
- Require interpretation and sense-making
- Value lies in early action advantage

### Readiness Dimensions

**Cultural**: Mindsets and behaviors that enable detection
**Structural**: Organizational designs that support information flow
**Cognitive**: Thinking capabilities for interpretation
**Resource**: Time, people, tools, and budget allocated

### Maturity Levels

**Level 1 - Initial**: Ad-hoc, no systematic approach
**Level 2 - Developing**: Some awareness, informal practices
**Level 3 - Defined**: Established processes, inconsistent application
**Level 4 - Managed**: Systematic approach, regular practice
**Level 5 - Optimizing**: Continuous improvement, integrated culture

### Priority Levels

**Immediate Action**: Fast-track review, form team, allocate resources
**High Priority**: Deep analysis, stakeholder engagement, pilot
**Monitor Actively**: Regular monitoring, expand sources, build capabilities
**Track & Review**: Add to radar, set triggers, share knowledge

## Implementation Best Practices

### Start Small
- Begin with 15 minutes weekly per person
- Pilot in receptive units first
- Choose 2-3 simple detection methods initially
- Demonstrate value before scaling

### Build Psychological Safety First
- Most critical cultural foundation
- Nothing else works without it
- Leadership must model vulnerability
- Reward signal sharing regardless of outcome
- Separate detection from judgment

### Focus on Strategic Relevance
- Not all signals deserve attention
- Filter for potential strategic impact
- Use prioritization frameworks consistently
- Resource constraints force choices

### Connect to Existing Processes
- Don't create separate bureaucracy
- Integrate with strategic planning
- Link to risk management
- Connect to innovation pipeline
- Part of normal business rhythm

### Celebrate Learning
- Share "interesting failures"
- Learn from missed signals
- Reward curiosity and questioning
- Make signal sharing visible
- Create signal success stories

## Common Pitfalls to Avoid

1. **Skipping readiness assessment** - Starting methods before building foundations
2. **Ignoring culture** - Focusing only on methods and tools
3. **Creating bureaucracy** - Overly complex processes that burden people
4. **Lack of prioritization** - Treating all signals equally
5. **No action linkage** - Signals don't connect to decisions
6. **Insufficient psychological safety** - People afraid to share
7. **Siloed scanning** - Missing cross-functional patterns
8. **One-time effort** - Not building sustainable capability
9. **Analysis paralysis** - Over-analyzing without acting
10. **Ignoring false signals** - Not learning from mistakes
